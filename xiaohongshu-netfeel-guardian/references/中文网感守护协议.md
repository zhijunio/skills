# 中文网感守护协议 v1.0

**适用场景**：任何使用 Claude 生成小红书、公众号、视频脚本、种草笔记等中文内容创作任务时，强制要求纯中文思维链 + 真实网感输出，防止英文思考 / 翻译腔 / 西方框架污染。

## 核心规则（必须严格执行）

1. **思维链语言强制**：
   - 整个规划、思考、举例、情绪判断过程**必须全程使用纯中文**。
   - 严禁在中间步骤使用英文思考再翻译输出。
   - 如果检测到英文思考痕迹，必须立刻打断并用中文重构。

2. **英文泄露 / 翻译腔检测清单**（每完成一段必须自检）：
   - 是否出现典型翻译句式：“亲爱的朋友们，你们能理解这种感受吗？”“In conclusion, we should...”“This is not just X, it's about Y”
   - 是否使用西方文化默认假设（美剧梗、个人主义英雄叙事、线性逻辑、过度正式的“conclusion”）
   - 情绪是否被稀释（真实小红书种草通常更直接、更口语化、带强烈情绪共鸣）
   - 是否出现文化错位（举例用美剧、西方节日、英文品牌梗）

3. **重写规则**：
   - 检测到翻译腔或英文框架 → 立刻用真实高赞小红书/公众号语料重写该段。
   - 优先保留用户原本想表达的核心观点和结构，只替换语言和文化框架。
   - 注入真实平台高互动表达（“姐妹们谁懂啊”“绝绝子”“泪目”“yyds”等自然口语，但需适度，避免过度）。

4. **平台适配（必须区分）**：
   - **小红书**：更情绪化、更口语化、带强烈共鸣和场景感（“姐妹们谁懂啊”“绝绝子”“泪目”）。
   - **公众号**：可稍正式，但仍要有人味、情绪曲线和真实故事感，避免纯鸡汤翻译体。
   - **视频脚本**：更短、更抓人、适合口播，增加停顿感和情绪起伏。

5. **最终自检（输出前必须再跑一遍）**：
   - 再次过检测清单。
   - 确认没有明显英文思考痕迹。
   - 输出“网感评分 + 英文泄露片段标注 + 3 个不同情绪强度的重写版本”。

## 与其他 Skill 的组合建议

- 推荐顺序：先完成平台框架与硬约束（标题、分段、CTA 等），再应用本守护协议做网感强化。
- 可选：用户明确要求时，再做一层去 AI 味润色（单独任务，不在本协议内自动跳转）。
- 所有写作类 Skill 都应在关键生成步骤后调用本守护协议。
