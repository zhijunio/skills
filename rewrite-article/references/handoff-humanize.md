# Post-structure polish (de-AI)

**本 skill** owns **structure and in-section clarity**. A separate **de-AI / register polish** pass owns wording, tells, bilingual layout, and copy modes — without reordering sections.

## 1. When to stop structure work

Stop structure work and offer polish when **all** are true:

1. Section order is **confirmed** and rewritten.
2. Wrap-up read in [`transitions-wrapup.md`](transitions-wrapup.md) is done.
3. User wants **de-AI**, **polish**, **release/social/bilingual** treatment, or says “润色” / “去 AI 味”.

Do **not** start polish mid-outline or mid-section unless the user explicitly skips structure work.

## 2. What structure work leaves for polish

| Already settled (do not redo in polish unless user asks) | Polish should focus on |
|----------------------------------------------------------|-------------------------|
| Section order and merges | Word-level tells, rhythm, register |
| Paragraph splits for length | ZH/EN spacing, banned phrases |
| DAG-fixed terminology consistency | Audience-appropriate tone; **do not compress §** |
| Factual content and stance | Mode-specific checklists (release, social, bilingual) |

Tell the user (one sentence, only if natural): structure is fixed; next step is a **de-AI polish pass** if they want one. Do **not** load another skill router — wait for the user to start a new task if they want dedicated polish tooling.

## 3. How to run polish (same session, only if user asks)

1. Use the **latest full revised text** (path or paste).
2. Polish **in place** — do **not** reorder sections or change heading levels unless the user asks.
3. Do **not** change facts “to sound better”; flag ambiguity and ask the user.

## 4. Scope boundaries

| User need | Action in **this skill** |
|-----------|---------------------------|
| Structure messy | Finish DAG reorder + section rewrite here first |
| Light touch-up, **no** reorder | User may skip this skill; do not force structure work |
| Research-from-scratch long-form | Out of scope — user starts a separate research workflow |

## 5. Anti-patterns

| Do not | Do instead |
|--------|------------|
| Apply de-AI word lists during section rewrite | Structure first; polish second (only if user asks) |
| Re-run full DAG reorder during polish | User must ask for structure again in a new pass |
| Silently invoke another skill for polish | Ask user; describe next capability in plain language |
