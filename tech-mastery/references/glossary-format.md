# Glossary format

File: **`<slug>-glossary.md`**. Recommended for mechanism, protocol, and spec topics.

Adapted from [mattpocock/skills teach GLOSSARY-FORMAT](https://github.com/mattpocock/skills), MIT.

## Template

```markdown
# {Topic} glossary

## Terms

**Term**:
One or two sentences — what it IS.
_Avoid_: confusing alias

**Another term**:
…
_Avoid_: …
```

## Rules

- **One workspace vocabulary** — main article, side articles, and interview use glossary terms.
- **Avoid aliases** — pick one term; list misleading alternatives under _Avoid_.
- **Tight definitions** — not how-to prose.
- **Revise in place** when understanding deepens on refresh.
- Add terms when the topic **uses** them consistently, not on first mention alone.
