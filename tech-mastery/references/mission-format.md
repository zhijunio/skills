# Mission format

Mission lives in **`<slug>-mastery-map.md` → ## Mission** (adapted from [mattpocock/skills teach MISSION-FORMAT](https://github.com/mattpocock/skills), MIT).

## Sections

| Section | Rule |
|---------|------|
| **Why** | 1–3 sentences. Concrete outcome when the topic is mastered — not「理解 X」. |
| **Success looks like** | Observable **reader** outcomes (explain, demo, interview). Not run meta (humanize, file checklist). |
| **Out of scope** | Protects boundary; list adjacent topics deferred on purpose. |
| **Constraints** | Optional: version, runtime, language, time, 简要/标准/深入. |

## Rules

- **Push back on vagueness.** If Why is abstract, ask before Source.
- **Revise on refresh** when the user's goal shifts; note in a learning record.
- **Keep short.** If Mission exceeds one screen, scope is probably too broad — split the topic.
